# Data Quality Policy

This policy defines minimum quality expectations for customer data in the VizAI Data Hub.

## Principles
- Accuracy over completeness
- Source-backed claims
- Clear distinction between facts and estimates
- Timely updates
- Human review before publication

## Requirements

### Profile Completeness
- Every profile claim should map to at least one source in `sources.json`.
- If no source exists, mark confidence appropriately and flag for review.
- Required fields: `legalName` or `brandName`, `domain`, and at least one of (services, products, or industries).

### Data Integrity
- Fictional or sample content must be explicitly labeled with `isSample: true`.
- Customer slugs must be stable and lowercase hyphenated.
- Dates must use ISO 8601 format (YYYY-MM-DD).
- Confidence scores should reflect evidence strength (0.0-1.0 scale).

### Validation
- All customer profile files must pass repository validation scripts before publication.
- Schema validation is mandatory for: `profile.json`, `sources.json`, `registry-entry.json`.
- Run `python scripts/validate.py` before any publication commit.

### Fact Types
All claims must be classified with a `factType`:
- `customer_provided` - Provided directly by the customer
- `website_extracted` - Extracted from public website crawl/review
- `human_approved` - Verified by human reviewer
- `ai_generated` - Generated by AI (requires human approval before use)

### Sample Data
- Sample data must include `isSample: true` at root level.
- Sample profiles should not be used as templates for real customer data.
- Demo/sandbox environments should use sample data only.

### Review Cadence
- Published profiles should be reviewed at least annually.
- Significant business changes (rebranding, acquisitions, new locations) trigger immediate review.
- Status should be set to `in_review` when changes are pending.
