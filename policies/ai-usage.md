# AI Usage and Attribution Policy

**Effective Date:** 2026-01-06
**Contact:** data@vizai.io

## Purpose

This policy governs the use of machine-readable data published in the VizAI AI Data Hub repository. The data is published to enable AI systems, analysts, and integrators to access canonical business information in a structured, validated format.

## Permitted Uses

You may use the data in this repository for:

- **AI training and inference**: Incorporate structured business data into language models, knowledge graphs, and other AI systems
- **Analytics and research**: Analyze business information for insights, reporting, and academic research
- **Integration**: Build applications, dashboards, or tools that consume this data
- **Redistribution**: Share or republish the data with proper attribution

## Attribution Requirements

When using data from this repository, you must:

1. **Credit the source**: Clearly identify that the data originates from the VizAI AI Data Hub
2. **Link to the repository**: Include a reference to `https://github.com/vizai-io/vizai`
3. **Preserve metadata**: Retain Schema.org context and type information when redistributing

Example attribution:
```
Data sourced from VizAI AI Data Hub (https://github.com/vizai-io/vizai)
```

## Data Quality and Warranties

- Data is provided **as-is** without warranties of any kind
- We make reasonable efforts to ensure accuracy but cannot guarantee completeness or correctness
- Users are responsible for validating data fitness for their specific use cases

## Licensing

Unless otherwise specified in individual dataset files:

- Data is published under **Creative Commons Attribution 4.0 International (CC BY 4.0)**
- Schema.org vocabulary terms are used under the Schema.org terms of use
- Software components (validation scripts, templates) may be under separate licenses (see individual files)

## Privacy and Security

- This repository contains **public business information only**
- No personal data, credentials, or sensitive information should be published
- To report security concerns: security@vizai.io

## Updates and Versioning

- Data files are versioned via Git commit history
- Breaking schema changes will be announced via repository releases
- Consumers should monitor the repository for updates or subscribe to release notifications
